# プロフィール — 作成するときのあなた自身

**プロフィール**とは、Lollyが「その人として」作成を行う際の作業上のアイデンティティです。氏名、連絡先、任意のヘッドショット、いくつかの環境設定など、毎回入力し直さずに済むようツールが参照できる小さな情報の集まりであり、それに加えて作業中に蓄積されるすべてのもの——保存済みセッション、アップロードした画像、ローカルの利用統計——も含まれます。

プロフィール内のすべては**デバイス上**、つまりブラウザのローカルデータベース（Web PWAではIndexedDB、Tauriアプリではファイルシステム）に保存されます。アカウントは存在せず、何もアップロードされることはありません。管理はギャラリー右上の**プロフィール**から行います。ツールはプロフィールを*読み取る*だけであり、しかもあらかじめ事前入力するよう作られた特定のフィールドに限られます。

> プロフィールは*あなた*（あるいはここで作成している人）についてのものです。ブランドの色やフォント、全体設定である**プラットフォーム**や、アプリでできることのカタログである**機能**とは異なります。詳しくは末尾の[プロフィール・プラットフォーム・機能の違い](#profile-vs-platform-vs-capabilities)をご覧ください。

## プロフィールに含まれるもの

| 項目 | 内容 |
|---|---|
| **氏名** | 姓と名。 |
| **連絡先** | メールアドレスと電話番号。 |
| **所在地** | 市区町村と国。 |
| **ヘッドショット** | 任意の写真で、正方形にトリミングされローカル画像として保存されます。メール署名、名言カード、カラーブロック、動的レイアウトなどのツールで使用されます。 |
| **自分の情報を使用** | 単一のオプトインスイッチです。個人情報が**プロヴェナンス**（書き出したファイルに埋め込まれる作成者／クレジット情報）として、また**/pro**の一括処理における作成者情報として付随するかどうかを制御します。（事前入力の可否を制御するものではありません。詳しくは[ツールがプロフィールをどう使うか](#how-tools-use-your-profile)をご覧ください。） |
| **環境設定** | テーマ（ライト、ダーク、またはSUSE）と、**機能フラグ**で有効にしたアプリの各部分。 |
| **あなたの作業** | 保存済みセッション（サムネイル付き）——**[プロジェクト](/info/using.html)**内のネストされたフォルダに整理されます——**マイ画像**ライブラリ、そしてこのプロフィールに紐づくローカルの利用統計。 |

これらはすべて任意です。空欄のプロフィールもそれはそれで問題なく機能します。入力の手間が省ける項目だけを埋めれば十分です。

## プロフィールは単なる「人」ではなく「文脈」

「プロフィール」という言葉は一人の固定された人物を連想させますが、Lollyにおいてそれは実際には**制作の文脈**——*何かを作っている間のあなたが誰であるか*——を意味します。この文脈には3つの異なる形があり、Lollyはそのすべてを同じ方法で扱います。

### 個人として

これが既定の形です。プロフィールはあなた自身——氏名、メールアドレス、ヘッドショット——を表します。一度設定すれば、署名、バッジ、カンファレンス用ロックアップがすべて自動的に入力されます。ほとんどの人にとってはこれで十分でしょう。

### チームとして

プロフィールは一人の人間である必要はありません。**組織内のチームや機能**——チームの共有名、グループの受信アドレス（`events@…`）、部署、チームのヘッドショットやユニットマークなど——を表すこともできます。一人が設定してエクスポートし（後述）、チームの他のメンバーが同じプロフィールを読み込めば、誰も入力し直すことなく、チームが作るものすべてに一貫した情報が付与されます。共用のキオスク端末や貸し出し用デモノートPCでも、その背後にいる全員が同一のチームプロフィールとして作成を行うことができます。

### 機能として——時々身にまとう役割

これは「一人につき一プロフィール」という硬直的なモデルでは捉えきれないケースです。あなたは年に3日だけ**イベントマネージャー**であり、それ以外の日はまったく別の何かかもしれません。その3日間はイベントの詳細、イベント用の受信アドレス、場合によってはバッジやサイネージに反映するイベントのサブブランドが欲しくなりますが、残りの362日は普段のアイデンティティに戻りたいはずです。

Lollyでは、その役割は単に**手元に置いておく別のプロフィール**——イベントのために読み込み、終わったら片付けておく保存済みバンドル（次のセクション参照）——にすぎません。役割は帽子であって、新しいアカウントではありません。必要なときにかぶり、終わったら脱げばよいのです。

## インストール1つにつきアクティブなプロフィールは1つ——保持できる数に制限はありません

どの瞬間においても、インストールには**アクティブなプロフィールが1つ**だけ存在します——それが今ツールから見えている情報です。アプリ内にプロフィール切り替え機能はありません。その代わり、各プロフィールは**持ち運び可能なバンドル**（1つの`.zip`ファイル、[下記](#moving-a-profile-to-a-new-device)を参照）です。これは新しいデバイスへ移行する際の仕組みとまったく同じで、意図的にそうなっています——プロフィールとは、保存し、コピーし、読み込むことができる1つのファイルなのです。

そのため、複数の文脈（あなた自身、チーム、イベントマネージャーという役割）を実際に使い分けている場合は、複数のバンドルを保持しておき、必要なものを読み込みます。

- **最もクリーンな切り替え方法：** **プロフィール → ストレージ → 自分のデータをすべて削除**を実行し、その後、これから入る文脈のバンドルを**インポート**します。これでそのプロフィールとしてのみ作成する状態になります。
- **重ね合わせ：** 先に削除せずにインポートすると**マージ**されます——インポートしたプロフィール、セッション、画像は既存のものの上に重ねられ、同名のものは置き換えられ、それ以外はそのまま残ります。あるチームの保存済みセッションを自分の環境に取り込みたいときには便利ですが、役割をきっちり区切りたい場合には向きません。
- **並行運用：** すべてがデバイス単位で管理されるため、別のブラウザプロフィール、別のユーザーアカウント、あるいは2つ目としてインストールしたPWAは、それぞれ独立したLollyプロフィールを持ちます。個人用インストールとイベント用キオスクインストールを切り替えることなく同時に実行できます。

> 文脈ごとにバンドルを保管し、内容がわかるようファイル名を変更しておきましょう（`LollyTools-events-2026.zip`、`LollyTools-me.zip`）。ファイルそのものが、プロフィール*なのです*。

## プロフィールを新しいデバイスへ移行する

プロフィールは完全にローカルなものであるため、まっさらなインストール——新しいノートPC、リセットしたばかりのブラウザ、同僚のマシン、オフラインの端末——にそれを持ち込む唯一の方法は、**ファイルを運ぶこと**です。ログインによって復元されることはありません。それこそが重要な点であり、そもそもあなたのデバイスから何かが外に出たことは一度もないのです。

**プロフィール → ストレージ → 別のデバイスへ移行**では、次のことができます。

- **自分のデータをエクスポート**を選ぶと、`LollyTools-<First>-<Last>-<YYYY-MM-DD>-<n>.zip`が1つダウンロードされます——ファイル名はそのプロフィールにちなんで付けられ、同日中に複数回エクスポートしても衝突しないよう連番が付きます（プロフィールに該当する情報がない場合、その部分は省略されます）。中身にはプロフィール本体、保存済みの全セッション（サムネイル付き）、アップロードした画像、そして各種設定（テーマ、レイアウト、ローカルの利用統計）が含まれます。
- 移行先のインストールで**データをインポート…**を選ぶとそのファイルが読み込まれ、まさに中断したところから作業を再開できます。

バンドルはそれ自体で完結した通常のzipファイルなので、USB、AirDrop、ネットワーク共有、自分宛のメールなど**あらゆる**手段で運ぶことができ、移行先は完全にオフラインでも構いません。各パートにはチェックサムが付いているため、転送中に破損したファイルは、中途半端に壊れた状態で復元されるのではなく、インポート時に検出されます。インポートは**マージ**方式（同名のプロフィール／セッション／画像は上書きされ、それ以外はそのまま保持）なので、すでに使用中の移行先を消去してしまうことはありません。

移行されないもの：カタログのキャッシュ（新しいデバイス上で自動的に再ダウンロードされます）と、ツール本体（すでに存在している前提です）。

バンドルの正確な構成、バージョンポリシー、整合性のルールについては**[データ移行](/info/data-transfer.html)**を、一連の流れを追った説明については**[Lollyの使い方 → 別のデバイスへ移行](/info/using.html#moving-to-another-device)**をご覧ください。

## ツールがプロフィールをどう使うか

ツールは、明示的に紐づけるよう作られたプロフィールのフィールドを*事前入力*するだけです。

**明示的な紐づけ。** ツールの作者は、入力項目がプロフィールから値を取得することを指定します（`bindToProfile: "firstname"`、`"email"`、`"headshot"`、…）。ツールを開くと、その入力項目はプロフィールから事前入力されます——それでいて、プロフィール自体を変更することなく、そのセッション限りで値を上書きすることもできます。事前入力はローカルな利便性のための機能であり、**自分の情報を使用**がオンかどうかに関わらず行われます。

**オプトイン（プロヴェナンス）。** アセットをエクスポートすると、あなたの情報は任意で**プロヴェナンス**——ファイルのメタデータ（PNG、PDF、SVGなど）に埋め込まれる作成者／クレジット情報——として付随し、完成したアセットが誰によって作られたかを示せるようになります。これを制御しているのが**自分の情報を使用**です。オフにしておくと、エクスポートには引き続き「Made with Lolly」というツール／プラットフォームのクレジットは含まれますが、個人の作成者／連絡先情報は埋め込まれません。（同じオプトインは**/pro**の一括処理における作成者情報にも適用されます。）（ツール作者の方へ：[ツールの作り方 → `bindToProfile`](/info/authoring-tools.html#bindtoprofile)および[ホストAPI → `host.profile`](/info/host-api.html#host-profile)をご覧ください。）

## プロフィール・プラットフォーム・機能の違い

この3つはUI上で近くに配置されており、混同しやすいものです。

- **プロフィール** — *あなた自身*（あるいはチーム、あるいは今担っている役割）：氏名、連絡先、ヘッドショット、保存済みの作業内容。個人的で、デバイスにローカルに保存され、バンドルとして持ち運び可能です。
- **プラットフォーム** — *ブランド*：すべてのツールが描画のよりどころとする色、フォント、全体設定。個人のものではなく、共有され一貫しています。
- **機能** — *アプリでできること*：利用可能な全機能セットとツール一式。

プロフィールはアセットが*誰から*のものかを変え、プラットフォームはそれが*どう見えるか*を変え、機能は*何を作れるか*を決めます。

## プライバシー

プロフィールが送信されたり、アップロードされたり、あなたを識別・追跡するために使われたりすることは一切ありません——同意すべきことは何もなく、何が保持されているかを知っていただくための、この告知があるのみです。**プロフィール → 自分のデータをすべて削除**からいつでもすべてを消去できます。詳しくは[プライバシーポリシー](/info/privacy.html)をご覧ください。
