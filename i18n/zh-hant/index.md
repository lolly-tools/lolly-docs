
# Lolly — 登陸頁文案

幾秒鐘內就能拿到可直接使用的檔案——設計規則、版面邏輯與輸出品質全部內建。不用等待,不用猜測。
在你自己的裝置上執行。不需要上傳,不需要學習,離線也能使用。

## 行銷人員

### 一切完成,不必等待。

你需要一張語錄卡、一張活動宣傳圖、一個在地化的簽名檔——今天就要,不是下一個衝刺週期。Lolly 會把完成、符合品牌規範的成品交給你,就算你從沒打開過設計工具也一樣。規則寫在範本裡,所以輸出結果一定正確。

- **填幾個欄位,就能拿到完成的素材。** 不用挑字型,不用猶豫顏色對不對,也不用擔心「這是正確的標誌嗎?」。工具早就知道答案,你只要提供文字內容。
- **無限排列組合,單一真實來源。** 把一張語錄卡在地化成 12 種語言、在 40 個活動中替換品牌組合標誌,或為每個版位產生所有尺寸變體——完全不用碰設計本身。全部都能用批次表格中的試算表驅動:貼上或匯入 CSV,每一列就會產生一個完成的素材。
- **例行素材不再受制於創意代理商瓶頸。** 電子郵件簽名檔、活動倒數計時、語錄卡、街道地圖——你的團隊在星期二下午四點需要的一切,自助完成。
- **協作,不必冒合規風險。** 沒有客戶資料上傳到第三方雲端服務,沒有 SaaS 授權需要稽核。完全在本機裝置運算。

## 記者

### 為速度與精準打造的編輯工具箱。

打造一次你的資訊編輯風格,之後就能根據即時資料,隨事件發生產生出版品質的素材。

- **把資料直接帶進視覺呈現。** 將結構化資料連接到圖表、地圖與表格範本。更新數字,版面自己會處理好。
- **精準吻合你的出版風格。** 工具在製作時就把字體排印與色彩限制寫死——你的內部風格規範在範本層級強制執行,而不是鬆散的指引。
- **可印刷或可上螢幕。** 從同一個範本輸出 SVG、CMYK PDF、高解析度 PNG,或 60fps 影片。單一來源,滿足製作部門需要的每一種格式。
- **可重複使用的格式,應付反覆出現的報導。** 選舉結果、季度財報、天氣事件——範本做一次,每次報導這類新聞都能重複使用。

## 營運

### 用工業級的嚴謹,產出日常內容。

不是所有東西都是行銷素材。托運標籤、識別證批量製作、合規單頁文件、事故通知——這些輸出的版面本身就是關鍵,資料絕不能出錯。Lolly 用處理頂尖設計同等的嚴謹,對待結構化內容。

- **輸入結構化資料,得到精確輸出。** 餵入一份 CSV 或 JSON 表格,每個欄位都會精準落在範本預期的位置——每一列產生一個完成的檔案。
- **實體尺寸精準。** 真實單位(mm/cm/in/pt)、真正的 DPI、出血與裁切線。標籤印表機或印刷廠拿到的正是它們需要的規格。
- **邏輯在工具內部執行。** 條碼、日期運算、條件式版面、對比檢查——都在渲染時計算完成,不需要手動處理。
- **設計上就是確定性的。** 相同輸入,永遠得到相同檔案。可稽核、可重複、可透過命令列或流水線自動化執行。

## 媒體與創意工作者

### 停止親手做機器最擅長的事。

你設計的是系統、解決的是問題,不是一次性成品。Lolly 是執行層,能把你的設計決策變成整個組織都能使用的工具——不需要你親自參與每一件素材的製作。

- **這是最前沿的領域。** 進階濾鏡、合成、動畫、生成式設計——這個平台沒有上限。每一種網頁技術,範本作者都能取用。做到任何創意套裝軟體都不允許的程度——同時擁有達成完成品質所需的全部能力。
- **工具組合工具。** 一個工具可以把另一個工具的輸出,當作即時素材嵌入——例如一張會自己產生 QR code 的識別證、一張直接放入即時圖表的卡片——不需要設計工具,也不需要手動合成。
- **製作工具,而不是製作檔案。** 打造一個能產生一萬張社群卡片的範本,而不是親手做出一萬張社群卡片。你的時間花在設計問題上,而不是生產流程上。
- **消除瑣碎的重複工作。** 最佳排版、標誌擺放位置、地圖圖釘定位、色彩對比檢查、輸出尺寸——全部在範本層級解決。你只要定義一次規則,重複的工作交給機器。
- **鎖住不該變動的,釋放該變動的。** 把品牌限制寫死,只公開真正該變動的變數。工具就成了創意的護欄。

## 開發者

### 你的建置流程,應該把視覺素材也包含進去。

圖片就是建置產物,應該這樣看待它。Lolly 可以從命令列執行,讓你用產生其他一切東西的相同方式來產生素材——可重複、自動化,並且是工作流程的一部分。

- **跳過生成式 AI 媒體的抽獎遊戲。** 你試過生成式 AI 媒體——昂貴,而且結果總是差那麼一截。Lolly 是免費、更快、在地端執行的內容自動化,去除了它們的粗製濫造,也省去你的苦工。
- **可重現的輸出。** 相同輸入永遠產生相同檔案。提交一個 URL,需要時就能重新產生——不用再把圖片檢入版本控制,也不用再去 Slack 追最新版本。
- **把媒體檔案移出你的程式碼庫。** 在建置時產生 OG 圖片、QR code、社群卡片與資料視覺化,而不是把二進位檔案存進 Git。
- **在素材內部執行邏輯。** 像 Code Canvas 和 Chart Creator 這類工具,讓你把真實內容——程式碼片段、結構化資料、即時數值——放進完成品質的範本裡,不需要自己打造渲染器。
- **零綁定。** 開放原始碼引擎、在地端運算、不需要 API 金鑰、沒有速率限制。

```bash
lolly qr-code --url=https://suse.com --output=og-qr.svg
lolly quotes --quote="Ship it." --output=quote.png
```

## AI 代理

### 讓你的模型使用工具,而不是產生幻覺。

一個帶參數的 URL 只需要幾個 token。一份創意簡報加上圖片生成則要花上數千個 token——而且結果還達不到出版品質。Lolly 為你的代理提供一個確定性、可審查、可直接用於正式產出的創意層。


- **正式產出品質不會飄移。** 工具用一小部分的運算量就能產出正式品質的成品——而且在本機完成。工具是寫死的。就算你的模型偷懶,版面也不會。字體排印規則、色彩數值、間距——這些都是結構性的,不是靠提示詞產生的。
- **大規模節省 token。** 用一個 URL 產生客製化的活動卡片,成本只是透過生成式模型以提示工程做出相同結果的一小部分。
- **把資料放在該放的地方。** 結構化輸入對應到結構化範本。講者姓名、場次時間、產品版本——每一次都精準放在設計預期的位置。
- **確定性、可稽核、可版本控制。** 每一個輸出都能從其輸入重現。正式素材中不會有隨機性的意外。

```
Use Lolly to invite the team to KubeCon.

Parameters:
  title: "KubeCon 2026"
  date: "2026-11-10"
  location: "Atlanta"

Output the file URL.
```

## 大型企業的 IT 與資安團隊

### 創意檔案不該是資安風險。

每次有人把檔案上傳到第三方服務,只是為了「讓它看起來對」,那就是一次等著發生的資料外洩事件。Lolly 從源頭消除這個問題——創意產出留在裝置端、留在你的內部環境,完全在你的掌控之中。

- **資料永不離開裝置。** 預設情況下,不會有任何資料傳送給第三方——沒有雲端渲染,沒有分析追蹤,沒有遙測。同事建立的內容留在自己的機器上。像「Strip Hidden Data」這類在地端工具,完全在瀏覽器內清除圖片、SVG 與 PDF 的 EXIF 與中繼資料——不需要上傳任何東西。就連為 PDF、整個下載檔案,或分享連結加上密碼保護,也是在裝置端完成;密碼永遠不會離開裝置。
- **自行架設,完全掌控實體隔離環境。** 部署在你自己的基礎設施上。沒有伺服器端處理,沒有資料庫,沒有後端。完全離線運作,在你的防火牆之後,套用你自己的存取政策。
- **縮小供應商暴露面。** 一個開放原始碼平台,取代一大堆用於創意產出的 SaaS 訂閱服務。供應商越少,代表合約越少、稽核越少,資安漏洞的入侵管道也越少。
- **在基礎設施層級落實品牌治理。** 工具只需製作一次,然後以資料形式散布——不是檔案,也不是人工流程。送出去的東西,正是核准要送出去的東西。
- **立即把關鍵資訊轉換成人類可讀的格式。** 事故通報、合規報告、高層簡報——結構化資料在幾秒內變成出版品質的輸出,沒有設計瓶頸。
- **為企業規模進行強化。** Lolly 是一個封閉試點的原型:其加密與檔案解析引擎目前正在接受 SUSE 嚴格的基礎設施強化,為企業規模做好準備——這件事我們非常擅長。來源證明與在地端加密在設計上就很強固;在強化完成期間,可以在限定範圍的情境中先行試點。詳見[採用與治理](/info/adoption-governance.html#status)。

## 工具


把它想像成一台設計自動販賣機。做出選擇,就能得到結果。每一次都是。

工具是一個現成可用的創意範本,清楚知道自己在做什麼、怎麼做。給它一個標題、一個日期、一個數字——它就會依照規格產出完成的素材。

不需要設計經驗。不需要決定字型、色彩或間距——工具早就知道最佳答案。不論你是設計師、行銷人員、開發者,還是 AI 代理,輸出結果都一樣。

工具由熟悉品牌的人製作一次,之後由所有其他人使用——透過網頁瀏覽器、手機、命令列,或自動化流水線。設計決策在範本層級就已經鎖定,剩下要做的只有填入內容。
