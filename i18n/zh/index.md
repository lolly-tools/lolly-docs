## 运营

### 为日常产出带来工业级的严谨。

并非所有工作都是市场营销。托运标签、批量胸卡、合规单页文件、事故通知——这些输出中排版承载着关键作用，数据绝不能出错。Lolly 以对待精英级设计同样的严谨态度处理结构化内容。

- **结构化数据输入，精确输出。** 输入一份 CSV 或 JSON 表格，每个字段都会精确落在模板预期的位置——每一行生成一份成品文件。
- **物理精度。** 真实单位（mm/cm/in/pt）、真实 DPI、出血线与裁切标记。标签打印机或印刷厂能获得它所需要的一切，分毫不差。
- **逻辑在工具内部运行。** 条形码、日期运算、条件排版、对比度检查——都在渲染时计算完成，无需手动处理。
- **设计上即具确定性。** 相同的输入，每次都会得到相同的文件。可审计、可重复，并可通过 CLI 或流水线实现自动化。

## 媒体与创意人员

### 别再做机器最擅长的事。

你设计的是系统、解决的是问题，而不是一次性的成品。Lolly 是执行层，它把你的设计决策转化为整个组织都能使用的工具——无需你参与每一份素材的制作。

- **这是前沿领域。** 高级滤镜、合成、动画、生成式设计——这个平台没有天花板。模板作者可以使用任何 Web 技术。做到任何创意套件都无法企及的程度——并拥有获得完成品级质量所需的全部能力。
- **工具组合工具。** 一个工具可以把另一个工具的输出作为实时素材嵌入其中——比如一张能渲染自己二维码的姓名牌，或一张能嵌入实时图表的卡片——无需设计工具，也无需手动合成。
- **创作工具，而非文件。** 构建一个能生成 10,000 张社交卡片的模板，而不是逐一制作 10,000 张卡片。你的时间花在设计问题上，而不是生产流程上。
- **消除繁琐工作。** 最优排版、标志摆放、地图图钉定位、色彩对比度检查、导出尺寸——全部在模板层解决。你只需定义一次规则，剩下的重复工作交给机器。
- **锁定不该变的，释放该变的。** 把品牌约束固化下来，只暴露真正需要变化的变量。工具由此成为创意的护栏。

## 开发者

### 你的构建流水线应该包含视觉素材。

图片就是构建产物，应当被这样对待。Lolly 可以从 CLI 运行，因此你可以用生成其他一切内容的同样方式来生成素材——可重复、自动化，并成为工作流的一部分。

- **跳过生成式 AI 媒体的赌博游戏。** 你尝试过生成式 AI 媒体——价格昂贵，效果也不到位。Lolly 免费、更快，是本地内容自动化方案，去掉了它们的粗劣产出，也省去了你的辛苦劳作。
- **可复现的输出。** 相同的输入每次都会生成相同的文件。提交一个 URL，按需重新生成——不再需要提交图片文件，也不用再去 Slack 上追最新版本。
- **把媒体文件从代码库中移出。** 在构建时生成 OG 图片、二维码、社交卡片和数据可视化图，而不是把二进制文件存进 Git。
- **在素材内部执行逻辑。** Code Canvas 和 Chart Creator 等工具让你可以把真实内容——代码片段、结构化数据、实时数值——放入具备生产级质量的模板中，而无需自建渲染器。
- **零锁定。** 开源引擎、本地计算、无需 API 密钥、没有速率限制。

```bash
lolly qr-code --url=https://suse.com --output=og-qr.svg
lolly quotes --quote="Ship it." --output=quote.png
```

## AI 智能体

### 让你的模型使用工具，而不是产生幻觉。

一个带参数的 URL 只需要几个 token。而一份创意简报加上图像生成则要耗费数千个 token——得到的结果甚至还达不到出版级质量。Lolly 为你的智能体提供了一个确定性的、可审查的创意层，随时可用于生产。


- **生产级质量不会漂移。** 工具以极低的算力就能产出生产级质量的作品——且在本地完成。工具是硬编码的。即使你的模型偷懒，排版也不会。字体排印规则、色彩数值、间距——它们是结构性的，而非靠提示词维持的。
- **大规模节省 token。** 通过 URL 生成一张自定义活动卡片的成本，只是通过生成式模型对同样输出做提示词工程所需成本的一小部分。
- **让数据各归其位。** 结构化输入映射到结构化模板。演讲者姓名、场次时间、产品版本——每一次都精确地落在设计预期的位置。
- **确定性、可审计、可版本控制。** 每一份输出都可以从其输入复现。生产素材中不会出现随机性的意外。

```
Use Lolly to invite the team to KubeCon.

Parameters:
  title: "KubeCon 2026"
  date: "2026-11-10"
  location: "Atlanta"

Output the file URL.
```

## 大型企业的 IT 与安全团队

### 创意文件不应该成为安全风险。

每当有人把文件上传到第三方服务，只是为了“让它看起来对一点”，这就是一次潜在的数据泄露事件。Lolly 从源头上消除了这个问题——创意制作始终留在设备上、留在本地，掌握在你自己手中。

- **数据永不离开设备。** 默认情况下，不会向任何第三方传输任何内容——没有云端渲染，没有分析统计，没有遥测数据。同事创建的内容始终留在他们自己的设备上。像 Strip Hidden Data 这样的设备端工具，完全在浏览器内清除图片、SVG 和 PDF 中的 EXIF 与元数据——不会上传任何内容。甚至为 PDF、整个下载文件或分享链接设置密码保护，也都在设备本地完成；密码永远不会离开设备。
- **自托管，实现完全的物理隔离控制。** 部署在你自己的基础设施上。没有服务器端处理，没有数据库，没有后端。完全离线运行，处于你的防火墙之后，遵循你自己的访问策略。
- **减少供应商暴露面。** 一个开源平台就能取代一大堆用于创意制作的 SaaS 订阅服务。更少的供应商意味着更少的合同、更少的审计，以及更少的泄露途径。
- **在基础设施层面强制执行品牌治理。** 工具只需编写一次，并作为数据分发——不是文件，也不是手动流程。最终对外发布的内容，正是被批准对外发布的内容。
- **即时将关键信息转化为易读格式。** 事故通报、合规报告、高管简报——结构化数据在几秒钟内就能变为出版级质量的输出，不再受制于设计瓶颈。
- **面向企业规模的加固。** Lolly 目前是一个封闭试点阶段的原型：其加密与文件解析引擎正在经历 SUSE 严格的基础设施加固，为迈向企业规模做准备——这方面我们确实很擅长。来源认证与设备端加密在设计上本就足够强大；在加固完成期间，可以在限定范围的场景中先行试点。参见[采用与治理](/info/adoption-governance.html#status)。

## 工具


可以把它想象成一台设计自动贩卖机。做出选择，得到结果。每一次都是如此。

一个工具就是一个即用型创意模板，它清楚地知道自己在制作什么、如何制作。给它一个标题、一个日期、一个数字——它就会按规格产出一份成品素材。

无需任何设计经验。无需决定字体、颜色或间距——工具早已了然于心。无论你是设计师、市场人员、开发者，还是 AI 智能体，得到的输出都是一样的。

工具由深谙品牌的人一次性编写完成，此后所有人都可以使用——无论是通过网页浏览器、手机、命令行，还是自动化流水线。设计决策已在模板层面锁定，剩下要做的只是填入内容。
