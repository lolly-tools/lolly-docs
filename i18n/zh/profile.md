# 个人资料——创作时你的身份

**个人资料**是 Lolly 创作时所采用的工作身份。它是一小组细节信息,工具可以从中调用,让你不必每次都重新输入——你的姓名、联系方式、可选的头像照片、一些偏好设置——再加上你在使用过程中积累的一切:已保存的工具会话、上传的图片,以及本地活动统计。

个人资料中的一切都保存在**本地设备**上,存放在浏览器的本地数据库中(Web PWA 上使用 IndexedDB,Tauri 应用中使用文件系统)。没有账户,也不会上传任何内容。你可以在**个人资料**(图库右上角)中管理它;工具只会*读取*它,而且只读取它们被设计用来预填的特定字段。

> 个人资料关乎*你*(或在此创作的任何人)。它不同于**平台**——品牌的颜色、字体和全局设置——也不同于**功能**,即应用程序能做什么的功能目录。参见文末的[个人资料 vs 平台 vs 功能](#profile-vs-platform-vs-capabilities)。

## 个人资料中有什么

| 部分 | 内容 |
|---|---|
| **姓名** | 名字和姓氏。 |
| **联系方式** | 电子邮箱和电话。 |
| **位置** | 城市和国家。 |
| **头像照片** | 一张可选的照片,裁剪为正方形并保存为本地图片。供电子邮件签名、引言卡片、色块和动态版面等工具使用。 |
| **使用我的详细信息** | 一个单一的启用开关。它控制你的个人详细信息是否作为**来源信息**——嵌入导出文件中的作者/署名信息——随行,以及是否作为 **/pro** 批量运行的作者。(它不影响预填:参见[工具如何使用你的个人资料](#how-tools-use-your-profile)。) |
| **偏好设置** | 你的主题(浅色、深色或 SUSE)以及你通过**功能开关**启用的应用部分。 |
| **你的作品** | 已保存的工具会话(带缩略图)——在**[项目](/info/using.html)**中整理为嵌套文件夹——你的**我的图片**素材库,以及本地活动统计,均与此个人资料关联。 |

以上都不是必填项。一个空白的个人资料本身就是完全可用的;你只需填写能替你省去重复输入的部分。

## 个人资料是一种情境,而不仅仅是一个人

“个人资料”这个词让人联想到一个固定不变的人,但在 Lolly 中,它其实是一种**创作情境**——*你在制作这件作品时是谁*。这种情境可以呈现三种不同的形态,Lolly 对待它们的方式完全相同。

### 作为个人

这是默认情况。个人资料就是你:你的姓名、你的邮箱、你的头像照片。设置一次,你的签名、徽章、会议标识组合就会自动填好。这是大多数人真正需要的全部。

### 作为团队

个人资料不一定代表单个人。它可以代表组织内的**一个团队或职能部门**:团队的共用名称、群组收件地址(`events@…`)、某个部门、团队的头像照片或单位标志。一个人设置好并导出它(见下文),团队其他人加载同一份个人资料——这样团队产出的一切都带有一致的信息,而不需要每个人重新输入。共用的展示终端或借出的演示笔记本电脑,也可以运行同一份团队个人资料,让在它背后创作的每个人都以此身份创作。

### 作为一种职能——你偶尔扮演的角色

这正是死板的“一人一份个人资料”模式所忽略的情况。你可能**一年中有三天是活动经理**,其余时间则完全是另一个身份。这三天里,你需要活动的详细信息、活动收件地址,或许还有一个活动子品牌来填充你的徽章和标识牌;其余 362 天,你想恢复平时的身份。

在 Lolly 中,这个角色只是**另一份随手可用的个人资料**——一个已保存的资料包(见下一节),你在活动时加载它,结束后放到一边。这个角色是一顶帽子,而不是一个新账户。需要时戴上,用完就摘下。

## 一次安装,一个活跃的个人资料——但你可以保留多份

在任意时刻,一次安装只有**一个活跃的个人资料**——也就是工具此刻能看到的信息。应用内没有个人资料切换器;相反,每份个人资料都是一个**可移动的资料包**(一个 `.zip` 文件,参见[下文](#moving-a-profile-to-a-new-device))。这是刻意设计成与更换设备相同的机制——个人资料就是一个你可以保存、复制和加载的文件。

所以,如果你确实需要在多个情境之间切换(你自己、你的团队、活动经理这顶帽子),就保留多个资料包,需要哪个就加载哪个:

- **最干净的切换方式:** **个人资料 → 存储 → 清除我的所有数据**,然后**导入**你要切换到的情境对应的资料包。此后你就纯粹以那份个人资料的身份创作。
- **叠加方式:** 不先清除就直接导入会**合并**——导入的个人资料、会话和图片会叠加在已有内容之上,同名的会被替换,其余保留。适合把某个团队已保存的会话拉进你自己的环境;但如果你需要清晰的角色边界,这就不是你想要的方式。
- **并行方式:** 由于一切都限定在设备范围内,一个独立的浏览器个人资料、一个独立的用户账户,或另一次安装的 PWA,各自都带有独立的 Lolly 个人资料。你可以同时运行你的个人安装和活动展示终端安装,无需切换。

> 为每个情境保留一个资料包,并按其用途重命名文件(`LollyTools-events-2026.zip`、`LollyTools-me.zip`)。这个文件*就是*个人资料。

## 将个人资料迁移到新设备

由于个人资料完全保存在本地,要把它带到一次全新的安装——一台新笔记本电脑、一个刚重置的浏览器、同事的电脑、一台离线设备——唯一的方法就是**携带这个文件**。没有登录能替你恢复它,而这正是重点所在:从一开始就没有任何东西离开过你的设备。

在**个人资料 → 存储 → 移动到另一台设备**中:

- **导出我的数据**会下载一个 `LollyTools-<First>-<Last>-<YYYY-MM-DD>-<n>.zip` 文件——以其所属的个人资料命名,并带有按天区分的序号,避免重复导出时发生冲突(个人资料缺少的部分会在文件名中省略)。它包含你的个人资料、每一个已保存的会话(及其缩略图)、你上传的图片,以及你的偏好设置(主题、版面、本地活动统计)。
- **导入数据……**在另一台安装上读取该文件,你就能从离开的地方原样继续。

这个资料包是一个普通的、自包含的 zip 文件,因此可以通过**任何**方式传输——U 盘、AirDrop、网络共享、发邮件给自己——目标设备也可以完全离线。每个部分都经过校验,因此传输中损坏的文件会在导入时被发现,而不会以半损坏状态被恢复。导入会**合并**(同名的个人资料/会话/图片会被覆盖;其余全部保留),所以它绝不会清空一个已在使用中的目标设备。

不会随行的内容:目录缓存(会在新设备上自动重新下载)以及工具本身(假定已经存在)。

关于资料包的确切结构、版本策略和完整性规则,参见**[数据迁移](/info/data-transfer.html)**;完整的端到端操作演示,参见**[使用 Lolly → 移动到另一台设备](/info/using.html#moving-to-another-device)**。

## 工具如何使用你的个人资料

工具只会*预填*它被明确设计用来绑定的个人资料字段:

**显式绑定。** 工具作者会将某个输入项标记为从个人资料中取值(`bindToProfile: "firstname"`、`"email"`、`"headshot"`……)。工具打开时,该输入项会从你的个人资料中预填——你仍然可以在这一次会话中覆盖它,而不会改动个人资料本身。预填是一种本地便利功能,无论**使用我的详细信息**是否开启都会发生。

**该选项(来源信息)。** 导出资源时,你的详细信息可以选择性地作为**来源信息**随行——即嵌入文件元数据(PNG、PDF、SVG 等)中的作者/署名信息——这样,一份成品资源就能说明是谁制作的。**使用我的详细信息**控制的正是*这一点*:关闭它,导出内容仍会带有“Made with Lolly”的工具/平台署名,但不会嵌入个人作者/联系方式信息。(同一个开关也决定 **/pro** 批量运行中的作者信息。)(工具作者请参见[编写工具指南 → `bindToProfile`](/info/authoring-tools.html#bindtoprofile) 和 [宿主 API → `host.profile`](/info/host-api.html#host-profile)。)

## 个人资料 vs 平台 vs 功能

界面中有三样东西彼此靠近,容易混淆:

- **个人资料** ——*你*(或你的团队,或你所扮演的角色):姓名、联系方式、头像照片、你保存的作品。个人化、保存在本地设备上、可以作为资料包移动。
- **平台** ——*品牌*本身:颜色、字体,以及每个工具据以渲染的全局设置。共享且一致,不具个人化。
- **功能** ——*应用程序能做什么*:你可用的完整功能集和工具。

个人资料决定一份资源*来自谁*;平台决定它*看起来是什么样*;功能决定*你能做出什么*。

## 隐私

个人资料从不会被传输、上传,也不会被用来识别或追踪你——没有什么需要你同意的,这条说明只是让你了解保留了哪些内容。你可以随时通过**个人资料 → 清除我的所有数据**将其全部清除。参见[隐私政策](/info/privacy.html)。
